Background

Thus far touch events have mainly only been supported on some mobile browsers. Firefox (as of version 4) is an exception in the desktop department and supports a mode where you can enable touch support (however it is a bit of a secret). Chrome's emulation of touch support, frankly just doesn't work properly.

For most of us this doesn't really cut it. I want multi-touch support on my desktop dammit! And across all browsers! This is what the aim of this library is. Using Caress Client + Caress Server + some hardware that spits out TUIO, you too can have multi-touch support in your browser!

Caress is a JavaScript library that converts TUIO events to native browser touches as proposed by the W3C Touch Events version 2.

Getting Started

If you have a Mac Trackpad you are in luck:

  1. Install NodeJS
  2. Download Tong Seng
  3. Grab the Caress Server or npm install caress-server
  4. cd /path/to/node_modules/caress-server
  5. npm install
  6. npm start
  7. Open and start Tong Seng
  8. Open http://localhost:5000/ in your favourite browser and check out the examples!

If you have an Android or iOS device, omit step 2 and:

  • Download TUIOpad and start stroking your iOS device!
  • or
  • Download TUIOdroid and rub your Android the right way!

Caressing Your Site

The Server

Grab the Caress Server, npm install caress-server and follow the instructions on the README.

The Client

Download the Caress Client. Then you just need to include a few client side libraries in your html file. They are:

            
              <script type="text/javascript" src="socket.io-0.9.10.min.js"></script>

              <script type="text/javascript" src="caress-0.1.0.js"></script>

Then initialize the client and start binding touch events at will!

            
              <script>
                $(function() {
                    window.client = new Caress.Client({
                        host: 'localhost',
                        port: 5000
                    });
                    client.connect();
                });
              </script>
